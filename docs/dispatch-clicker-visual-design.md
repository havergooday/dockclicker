# Dispatch Clicker 디자인 문서

## 개요

이 문서는 `Dispatch Clicker`의 와이어프레임과 시각적 방향을 정리한 디자인 문서다.  
게임은 스팀 출시를 목표로 하는 `데스크톱 컴패니언형 방치/클리커`이며, `화면 하단 가로형`과 `화면 측면 세로형`을 모두 지원하는 방향으로 설계한다.

최신 UX 재설계 방향은 [UI/UX 전면 재설계 계획](plans/2026-05-27-ux-redesign.md)을 기준으로 한다.
현재 확정 방향은 기존 패널 플립/운영 패널 중심 구조가 아니라, `좌측 격납고 / 중앙 브릿지·파일럿 라운지 / 우측 관제실`로 이어지는 파노라마 함선 캔버스와 카메라 이동 구조다.
현재 구현은 이 방향의 1차 단계로 `ShipCanvas`와 관제실 `항성지도` 팝업을 반영한 상태다.

## 비주얼 방향

- 전체 톤은 `SF 함선 내부` 기반으로 잡는다.
- 지나치게 무겁거나 군사적인 SF보다는 `생활감 있는 우주 함선` 느낌을 우선한다.
- 화면 중심은 `브릿지·파일럿 라운지`이며, 분위기와 감정 포인트는 `SD 미소녀 캐릭터`가 담당한다.
- 게임은 코지한 정서를 유지하되, 단순 장식 게임이 아니라 `파견 운영 대시보드`처럼 보여야 한다.

## 디자인 목표

- 업무 중 화면 한쪽에 띄워두기 쉬운 구조
- 한눈에 읽히는 핵심 운영 정보
- 캐릭터가 살아 움직이는 느낌
- 작은 화면에서도 정보가 뭉개지지 않는 밀도
- 장시간 켜두어도 피로하지 않은 UI

## 메인 와이어프레임 원칙

최신 방향에서는 중앙 운영 패널보다 `중앙 브릿지·파일럿 라운지`가 기본 홈이다.
운영 기능은 좌측 격납고와 우측 관제실에 나뉘어 배치하고, 조립/상점/고용은 각 구역의 팝업으로 호출한다.

- 중앙은 파일럿 라운지/브릿지 홈으로 두고, 파일럿과 꾸미기 보상이 항상 보이게 한다.
- 머신 관련 정보는 좌측 격납고에 둔다.
- 행성 개척과 파견 명령은 우측 관제실에 둔다.
- 조립 기능은 격납고 팝업, 상점/고용 기능은 관제실 팝업으로 호출한다.
- 옵션과 설정류는 화면 가장자리 글로벌 유틸 바에 둔다.
- 네비게이션 탭과 우클릭 드래그로 함선 내부를 이동한다.

## 가로형 메인 와이어프레임

가로형은 가장 많은 정보가 표시되는 기본 레이아웃이다.  
전체 구조는 `파노라마 함선 단면형`으로 잡는다.

```text
[격납고 확장 구역] ← [격납고] ← [브릿지/파일럿 라운지] → [관제실]
```

### 1. 좌측 구역: 격납고

- 파견 머신이 대기하는 공간
- 대기 중, 정비 중, 출격 준비, 복귀 완료 같은 상태를 표시
- 머신은 카드 또는 슬롯 형태로 정렬
- 상태등, 타이머, 간단한 경고 정보를 붙여 빠르게 읽을 수 있게 구성
- 베이 수가 늘어나면 좌측으로 공간이 확장된다
- 파츠 조립, 인벤토리, 함선 강화는 격납고 내부 팝업으로 호출한다

### 2. 중앙 구역: 브릿지/파일럿 라운지

- 기본 홈 화면
- SD 파일럿 캐릭터들이 머무는 공간
- 캐릭터는 걷기, 대화, 휴식, 가구 사용 같은 생활 연출을 가진다
- 꾸미기 가구와 함선 확장 보상이 실제 배경에 누적되어 보인다
- 글로벌 파견 요약은 작고 항상 보이는 HUD로 처리한다

### 3. 우측 구역: 관제실

- 항성지도 팝업 진입, 원정 편성, 파견 명령을 담당한다
- 현재 목표 원정, 다음 해금 목표, 파견 가능 베이 수를 보여준다
- 직접 출격과 자동 파견 명령은 관제실에서 시작한다
- 파일럿 고용, 파츠/가구 구매는 관제실 터미널 팝업으로 호출한다

### 4. 우측 끝 구역: 유틸 바

- 설정
- 로그
- 알림
- 사운드
- 속도 조절
- 최소화
- 운영과 직접 관련 없는 버튼은 모두 여기에 세로로 정렬한다

## 정보 흐름

플레이어는 아래 순서로 화면을 읽는다.

1. 좌측에서 `머신 상태`를 확인한다.
2. 중앙 홈에서 `파일럿 상태`와 함선 분위기를 확인한다.
3. 우측 관제실에서 `행성 개척`과 파견 명령을 처리한다.
4. 필요할 때 각 구역 팝업으로 조립, 구매, 고용을 처리한다.

이 구조를 통해 `내 함선 홈 - 머신 운용 - 행성 개척 - 재투자` 흐름이 자연스럽게 형성되도록 한다.

## 세로형 메인 와이어프레임

세로형은 현재 보류한다.
다만 향후 숙소/생활층을 아래 데크로 추가할 수 있도록 카메라 시스템은 X/Y 이동을 모두 지원할 수 있게 설계한다.

초기 구현에서는 가로형 파노라마를 우선한다.

## 가로형 / 세로형 비교

### 공통 원칙

- 두 레이아웃 모두 `파일럿 홈 → 격납고/관제실 → 팝업 조작` 흐름을 유지한다.
- 중앙 브릿지/파일럿 라운지가 감정적 기준점이다.
- 좌우 또는 상하의 생활 공간은 단순 배경이 아니라 실제 상태 정보를 담는다.
- SD 캐릭터는 감성 포인트이면서 동시에 플레이 상태를 전달하는 역할을 맡는다.

### 가로형 특징

- 가장 많은 정보를 표시하는 기본 레이아웃
- `좌측 격납고 / 중앙 브릿지·파일럿 라운지 / 우측 관제실` 구조
- 사용자는 화면을 `풍경처럼` 훑으며 상태를 확인한다
- 감성 연출과 운영 정보의 균형이 좋다
- 메인 모니터 하단이나 넓은 영역에 두기 적합하다

### 세로형 특징

- 좁은 화면 폭에 맞춘 보조 레이아웃
- 추후 아래 데크/숙소층 확장과 함께 재검토
- 사용자는 화면을 `카드 목록처럼` 위에서 아래로 읽는다
- 가독성과 핵심 숫자 우선 정리가 중요하다
- 보조 모니터 측면이나 메인 화면 한쪽에 두기 적합하다

### 역할 차이 요약

- 가로형은 `연출과 정보의 균형`이 강하다.
- 세로형은 `가독성과 압축된 운영 정보`가 강하다.
- 가로형은 메인 레이아웃이다.
- 세로형은 동일한 정보를 더 좁은 공간에 맞게 재배치한 대응 레이아웃이다.

## 캐릭터 활용 방향

- SD 미소녀 캐릭터는 화면의 감정 포인트이자 정보 앵커다.
- 캐릭터는 데스크톱 펫처럼 화면을 지배하지 않고, 운영 공간 안에서 기능적으로 존재해야 한다.
- 캐릭터별로 역할, 상태, 피로도, 배치 가능 여부가 다르게 보이게 한다.
- 장기적으로는 파일럿마다 다른 외형, 성향, 특기 연출이 가능하도록 확장 여지를 둔다.

## 색감 및 분위기

- 메인 색상은 `네이비`, `시안`, `화이트` 기반
- 포인트 색상은 `오렌지` 또는 `라임` 계열로 경고/활성 상태를 구분
- 중앙 패널은 홀로그램 HUD 같은 차갑고 정돈된 느낌
- 좌우 실내는 금속감보다는 생활감이 느껴지는 밝은 SF 인테리어
- 캐릭터는 귀엽지만 화면 전체가 지나치게 파스텔톤으로 흐르지 않도록 조절

## 글로벌 상태 표시줄 (Status Bar)

화면 한쪽 가장자리에 고정되는 항상 보이는 정보 바다.

### 표시 정보 (1차)
- 보유 크레딧 (CR 수치, 변경 시 트윈 애니메이션)
- 추후 추가 가능: 현재 파견 수, 귀환 대기 수, 경고 알림

### 위치 옵션 (설정에서 변경 가능 — 미구현)
- **하단** (기본): 전체 너비, 화면 맨 아래 고정
- **상단**: 전체 너비, 화면 맨 위 고정
- **좌측**: 전체 높이, 화면 왼쪽 세로 고정
- **우측**: 전체 높이, 화면 오른쪽 세로 고정
- 위치 설정은 `GameState.layout_mode` 또는 별도 `status_bar_position` 변수로 관리
- 위치에 따라 패널들의 여백도 자동으로 조정되어야 한다

### 구현 상태
- `scenes/ui/credit_hud.tscn` + `scripts/ui/credit_hud.gd`
- 화면 하단 **플로팅 pill 오버레이** — 공간 점유 없이 모든 패널 위에 띄움
- 현재 보유 CR 표시, 변경 시 숫자 트윈 애니메이션, 수령 시 +금액 날아오는 연출
- 위치 변경 기능은 미구현 (설정 UI 작업 시 함께 추가할 것)

## 디자인 메모

- 중앙 패널은 숫자와 버튼이 우선이다.
- 좌우 공간은 감성만이 아니라 상태 정보를 담아야 한다.
- 장시간 켜둘 게임이므로 애니메이션은 짧고 반복 피로가 적어야 한다.
- 세로형은 가로형의 대체물이 아니라 같은 정보 구조를 좁은 화면용으로 재배치한 대응 레이아웃이다.
