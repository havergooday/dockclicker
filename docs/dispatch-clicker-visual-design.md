# Dispatch Clicker 디자인 문서

## 개요

이 문서는 `Dispatch Clicker`의 와이어프레임과 시각적 방향을 정리한 디자인 문서다.  
게임은 스팀 출시를 목표로 하는 `데스크톱 컴패니언형 방치/클리커`이며, `화면 하단 가로형`과 `화면 측면 세로형`을 모두 지원하는 방향으로 설계한다.

## 비주얼 방향

- 전체 톤은 `SF 함선 내부` 기반으로 잡는다.
- 지나치게 무겁거나 군사적인 SF보다는 `생활감 있는 우주 함선` 느낌을 우선한다.
- 화면 중심은 운영 패널이지만, 분위기와 감정 포인트는 `SD 미소녀 캐릭터`가 담당한다.
- 게임은 코지한 정서를 유지하되, 단순 장식 게임이 아니라 `파견 운영 대시보드`처럼 보여야 한다.

## 디자인 목표

- 업무 중 화면 한쪽에 띄워두기 쉬운 구조
- 한눈에 읽히는 핵심 운영 정보
- 캐릭터가 살아 움직이는 느낌
- 작은 화면에서도 정보가 뭉개지지 않는 밀도
- 장시간 켜두어도 피로하지 않은 UI

## 메인 와이어프레임 원칙

- 운영 정보는 중앙에 모은다.
- 머신 관련 정보는 좌측에 둔다.
- 파일럿 및 생활 정보는 우측에 둔다.
- 옵션과 설정류는 가장 바깥쪽으로 밀어낸다.
- 좌우 공간은 단순 배경이 아니라 실제 게임 정보를 담는 공간으로 사용한다.

## 가로형 메인 와이어프레임

가로형은 가장 많은 정보가 표시되는 기본 레이아웃이다.  
전체 구조는 `중앙 통제실형`으로 잡는다.

### 1. 좌측 구역: 격납고

- 파견 머신이 대기하는 공간
- 대기 중, 정비 중, 출격 준비, 복귀 완료 같은 상태를 표시
- 머신은 카드 또는 슬롯 형태로 정렬
- 상태등, 타이머, 간단한 경고 정보를 붙여 빠르게 읽을 수 있게 구성

### 2. 중앙 구역: 운영 패널

- 게임의 핵심 조작과 숫자가 모이는 중심 패널
- 현재 파견 수
- 복귀 대기 수
- 정산 가능 수익
- 직접 파견 버튼
- 자동 파견 슬롯 상태
- 선택 지역 정보
- 가장 중요한 행동 버튼과 수치는 이 구역에만 둔다

### 3. 중앙 우측 구역: 브리지/오퍼레이션 라운지

- SD 파일럿 캐릭터들이 머무는 공간
- 캐릭터는 걷기, 대화, 커피 마시기, 대기, 휴식 같은 생활 연출을 가진다
- 동시에 `휴식 중`, `배치 가능`, `피로 높음` 같은 실제 상태 정보를 함께 보여준다
- 단순 장식 구역이 아니라 `파일럿 상태 표시 구역`의 역할을 가진다

### 4. 우측 끝 구역: 유틸 바

- 설정
- 로그
- 알림
- 사운드
- 속도 조절
- 최소화
- 운영과 직접 관련 없는 버튼은 모두 여기에 세로로 정렬한다

## 정보 흐름

플레이어는 아래 순서로 화면을 읽는다.

1. 좌측에서 `머신 상태`를 확인한다.
2. 중앙에서 `운영 판단`을 내린다.
3. 우측에서 `파일럿 상태`를 확인한다.
4. 필요할 때만 우측 끝 유틸 바를 조작한다.

이 구조를 통해 `장비 - 운영 - 인원 - 유틸` 순서가 자연스럽게 형성되도록 한다.

## 세로형 메인 와이어프레임

세로형은 가로형 구조를 유지한 채, 이를 `상하 재배치`하는 보조 레이아웃이다.  
전체 구조는 `상단 격납고 - 중단 운영 - 하단 브리지` 순서로 잡는다.

### 1. 상단 구역: 격납고 스트립

- 머신 상태를 먼저 확인하는 영역
- 세로 폭에 맞춰 1열 또는 2열 슬롯으로 구성
- 각 머신은 `대기`, `정비`, `출격 중`, `복귀 완료` 상태를 짧게 표시
- 긴 설명보다 상태등, 타이머, 간단한 아이콘 중심으로 구성

### 2. 중단 구역: 운영 패널

- 세로형에서도 가장 중요한 핵심 조작 구역
- 현재 수익
- 활성 파견 수
- 복귀 대기 수
- 직접 파견 버튼
- 자동 파견 슬롯 상태
- 선택 지역 정보
- 주요 버튼은 이 구역에 고정 배치

### 3. 하단 구역: 브리지/오퍼레이션 라운지

- SD 파일럿 캐릭터들이 머무는 공간
- 작은 화면에서도 캐릭터 존재감이 유지되도록 짧고 선명한 연출 사용
- `휴식 중`, `배치 가능`, `피로 높음` 같은 상태를 함께 노출
- 장식 배경이 아니라 실제 상태 확인 구역으로 사용

### 4. 유틸 영역

- 설정, 로그, 알림, 사운드, 속도 조절, 최소화 기능 배치
- 세로형에서는 상시 노출 비중을 줄이고 접이식 패널 또는 아이콘 바 형태를 우선 고려
- 운영 흐름을 끊지 않는 위치에 배치

## 가로형 / 세로형 비교

### 공통 원칙

- 두 레이아웃 모두 `머신 상태 → 운영 판단 → 파일럿 상태 → 유틸` 흐름을 유지한다.
- 중앙 또는 중단 운영 패널이 가장 중요한 조작 지점이다.
- 좌우 또는 상하의 생활 공간은 단순 배경이 아니라 실제 상태 정보를 담는다.
- SD 캐릭터는 감성 포인트이면서 동시에 플레이 상태를 전달하는 역할을 맡는다.

### 가로형 특징

- 가장 많은 정보를 표시하는 기본 레이아웃
- `좌측 격납고 / 중앙 운영 / 우측 브리지 / 우측 끝 유틸` 구조
- 사용자는 화면을 `풍경처럼` 훑으며 상태를 확인한다
- 감성 연출과 운영 정보의 균형이 좋다
- 메인 모니터 하단이나 넓은 영역에 두기 적합하다

### 세로형 특징

- 좁은 화면 폭에 맞춘 보조 레이아웃
- `상단 격납고 / 중단 운영 / 하단 브리지` 구조
- 사용자는 화면을 `카드 목록처럼` 위에서 아래로 읽는다
- 가독성과 핵심 숫자 우선 정리가 중요하다
- 보조 모니터 측면이나 메인 화면 한쪽에 두기 적합하다

### 역할 차이 요약

- 가로형은 `연출과 정보의 균형`이 강하다.
- 세로형은 `가독성과 압축된 운영 정보`가 강하다.
- 가로형은 메인 레이아웃이다.
- 세로형은 동일한 정보를 더 좁은 공간에 맞게 재배치한 대응 레이아웃이다.

## 캐릭터 활용 방향

- SD 미소녀 캐릭터는 화면의 감정 포인트이자 정보 앵커다.
- 캐릭터는 데스크톱 펫처럼 화면을 지배하지 않고, 운영 공간 안에서 기능적으로 존재해야 한다.
- 캐릭터별로 역할, 상태, 피로도, 배치 가능 여부가 다르게 보이게 한다.
- 장기적으로는 파일럿마다 다른 외형, 성향, 특기 연출이 가능하도록 확장 여지를 둔다.

## 색감 및 분위기

- 메인 색상은 `네이비`, `시안`, `화이트` 기반
- 포인트 색상은 `오렌지` 또는 `라임` 계열로 경고/활성 상태를 구분
- 중앙 패널은 홀로그램 HUD 같은 차갑고 정돈된 느낌
- 좌우 실내는 금속감보다는 생활감이 느껴지는 밝은 SF 인테리어
- 캐릭터는 귀엽지만 화면 전체가 지나치게 파스텔톤으로 흐르지 않도록 조절

## 글로벌 상태 표시줄 (Status Bar)

화면 한쪽 가장자리에 고정되는 항상 보이는 정보 바다.

### 표시 정보 (1차)
- 보유 크레딧 (CR 수치, 변경 시 트윈 애니메이션)
- 추후 추가 가능: 현재 파견 수, 귀환 대기 수, 경고 알림

### 위치 옵션 (설정에서 변경 가능 — 미구현)
- **하단** (기본): 전체 너비, 화면 맨 아래 고정
- **상단**: 전체 너비, 화면 맨 위 고정
- **좌측**: 전체 높이, 화면 왼쪽 세로 고정
- **우측**: 전체 높이, 화면 오른쪽 세로 고정
- 위치 설정은 `GameState.layout_mode` 또는 별도 `status_bar_position` 변수로 관리
- 위치에 따라 패널들의 여백도 자동으로 조정되어야 한다

### 구현 상태
- `scenes/ui/credit_hud.tscn` + `scripts/ui/credit_hud.gd`
- 현재 하단 고정 구현, 전체 너비 26px 높이
- 위치 변경 기능은 미구현 (설정 UI 작업 시 함께 추가할 것)

## 디자인 메모

- 중앙 패널은 숫자와 버튼이 우선이다.
- 좌우 공간은 감성만이 아니라 상태 정보를 담아야 한다.
- 장시간 켜둘 게임이므로 애니메이션은 짧고 반복 피로가 적어야 한다.
- 세로형은 가로형의 대체물이 아니라 같은 정보 구조를 좁은 화면용으로 재배치한 대응 레이아웃이다.
