# Dispatch Clicker 기획서

## 개요

`Dispatch Clicker`는 `파견`을 중심으로 돌아가는 방치형/클리커 게임이다.  
플레이어는 처음에 자동화 수단이 전혀 없는 상태에서 직접 파견 임무를 수행해 초기 자금을 확보한다. 이후 수익을 머신 파츠 구매, 파일럿 고용, 시설 확장에 재투자하면서 수동 플레이를 자동 파견 네트워크로 확장해나간다.

와이어프레임 및 화면 구성에 대한 내용은 [dispatch-clicker-visual-design.md](dispatch-clicker-visual-design.md)를 참고한다.

최신 UI/UX 방향은 [UI/UX 전면 재설계 계획](plans/2026-05-27-ux-redesign.md)을 기준으로 한다.
기존 패널 플립 탐색은 초기 구현 구조이며, 다음 구조는 `좌측 격납고 / 중앙 브릿지·파일럿 라운지 / 우측 관제실`로 이어지는 파노라마 함선 캔버스와 카메라 이동 방식이다.

## 핵심 판타지

- 플레이어는 초기에 유일한 운용 인력이다.
- 직접 파견되어 클릭으로 임무를 수행한다.
- 복귀 후 파견 성과와 시설 상태에 따라 수익을 얻는다.
- 얻은 수익으로 머신을 조립하고 파일럿을 고용한다.
- 완성된 파견 머신을 자동으로 운용한다.
- 기지와 시설을 확장해 여러 파견 유닛을 동시에 관리한다.

## 코어 루프

1. 지역을 선택하고 플레이어를 직접 파견한다.
2. 파견 중 클릭으로 임무를 수행한다.
3. 일정 시간이 지나거나 원하는 시점에 복귀한다.
4. 임무 결과와 시설 상태에 따라 수익을 획득한다.
5. 수익으로 머신 파츠, 업그레이드, 확장 요소를 구매한다.
6. 머신을 조립하고 파일럿을 배정한다.
7. 자동 파견 유닛을 지역에 보내 추가 수익을 얻는다.
8. 더 큰 시설, 더 많은 유닛, 더 강한 플레이어 성능에 재투자한다.

## 패널 구조 (공간 탐색 방식)

함선 메인 뷰 자체가 브리지 역할을 겸한다.  
SD 캐릭터들이 idle 상태로 돌아다니고, 파견 중 유닛 상태가 표시된다.

초기 구현은 각 구역의 인터랙션 오브젝트를 클릭하면 해당 패널로 플립되는 방식이다.
최신 UX 방향은 플립 대신 가로로 긴 함선 캔버스를 두고, 탭 클릭 또는 우클릭 드래그로 카메라가 `격납고 ↔ 브릿지/라운지 ↔ 관제실` 사이를 이동하는 방식이다.

| 오브젝트 | 플립 후 화면 | 주요 기능 |
|---|---|---|
| 격납고 문 | 격납고 패널 | 귀환 유닛 수령, 크레딧 획득, 머신 상태 확인 |
| 공작실 입구 | 공작실 패널 | 보유 파츠로 머신 조립 |
| PC 터미널 | 상점 패널 | 파츠 구매, 파일럿 고용 |
| 관제 콘솔 | 파견 관제 패널 | 파견 현황 확인, 유닛 배정, 전원 출격, 직접 출격 |

파견 관제 패널에서 파일럿마다 담당 행성을 지정해두면 복귀 후 자동 재출격한다.  
행성은 임무 보드(카드 목록) 형태로 표시되며, 크레딧으로 새 행성을 해금한다.

## 초반 구조

- 시작 시 플레이어 외에는 아무것도 없다.
- 초기 성장은 반복적인 수동 파견으로 이루어진다.
- 수동 파견은 가장 기본적인 클릭 플레이 구간이다.
- 첫 번째 목표는 `첫 파견 머신 제작`과 `첫 파일럿 고용`이다.

## 직접 파견 클리커 시스템

플레이어가 직접 파견을 나가면 패널이 뒤집히며 클리커 전투 화면으로 전환된다.  
함선 창 크기 그대로 유지되며 별도 창이나 전환 없이 패널 플립으로 처리한다.

### 기본 구조

- 클릭 1회 = 타겟에 데미지
- 외계생물은 HP를 가지며, 0이 되면 처치되고 보상(크레딧, 파츠 등)을 드랍한다
- 제한 시간 종료 또는 플레이어가 직접 복귀를 선택하면 임무가 끝난다

### 초반 ~ 후반 진행 흐름

| 단계 | 화면 구성 | 플레이 느낌 |
|---|---|---|
| 초반 | 생물 1~2마리, 느린 이동 | 여유롭게 한 마리씩 클릭 |
| 중반 | 여러 마리 동시 등장, 빠른 생물 혼합 | 우선순위 판단 필요 |
| 후반 | 무리 단위 등장, 강한 개체 포함 | 범위 클릭으로 무리 처리 |

### 플레이어 기체 업그레이드 축

- **클릭 데미지** — 클릭 1회당 데미지 증가. 가장 기본적인 수치 강화
- **클릭 범위** — 클릭 시 타격 범위 확장. 단일 타격 → 범위 타격으로 플레이 방식이 변화
- **무기 종류** — 무기 변경으로 데미지 유형이나 특성이 달라짐 (예: 관통, 폭발 등, 추후 설계)
- **행성/지역 해금** — 더 강한 생물, 더 좋은 보상이 있는 지역으로 출격 가능

### 지역(행성) 구조

- 초반에는 기본 행성 1개만 접근 가능
- 모은 크레딧을 지불해 새로운 행성을 해금한다 (크레딧 구매 방식)
- 지역마다 등장 생물 종류와 보상 품질이 다르다
- 고난이도 행성에서는 크레딧 외에 머신 파츠도 드랍된다
- 지역 특성은 추후 설계 (환경 효과, 특수 조건 등)

## 파견 시스템

- 파견은 게임의 기본 행동이다.
- 플레이어는 목표 지역을 선택하고 임무를 시작한다.
- 파견 중에는 클릭을 통해 임무를 수행한다.
- 임무는 `정해진 시간 종료` 또는 `플레이어 수동 복귀`로 마무리된다.
- 보상은 복귀 시 정산된다.
- 보상은 아래 요소의 영향을 받는다.
- `임무 수행 성과`
- `지역 특성`
- `머신 스탯`
- `시설 상태`

## 자동 파견 모니터링 (CCTV)

- 파일럿이 파견 중인 경우 함선 패널에 "파견 중" 상태로만 표시된다.
- 플레이어가 원하면 소형 CCTV 창을 열어 작업 상황을 엿볼 수 있다.
- CCTV 창은 선택 사항이며, 닫아도 파견 진행에는 영향이 없다.
- 리소스 부담을 줄이기 위해 풀 배틀 연출 대신 저해상도/노이즈 필터 느낌의 간략한 화면으로 구성한다.
- 구체적인 CCTV 화면 설계(상태바, 픽셀 뷰, 텍스트 피드 등)는 MVP 이후 단계에서 결정한다.

## 머신 및 파일럿 시스템

- 수익으로 머신 파츠를 구매하거나 고난이도 행성에서 드랍으로 획득한다.
- 구매한 파츠를 조립해 파견 머신을 제작한다. (공작실 패널에서 수행)
- 파일럿 고용과 머신 파츠 구매는 모두 PC 패널(상점 터미널)에서 수행한다.
- `머신 + 파일럿` 조합이 하나의 자동 파견 유닛이 된다.
- 자동 파견 유닛은 일정 시간 동안 지역에 파견된 뒤 복귀한다.
- 복귀 후에는 휴식, 정비, 재설정 등의 재가동 대기 시간이 필요하다.

### 파츠 아이템 모델 (설계 원칙)

파츠는 **수량 스택이 아닌 개별 아이템 인스턴스**로 취급한다.

- 같은 종류의 파츠라도 각각 고유한 인스턴스다.  
  예: "경량 프레임 2개"는 인스턴스 A와 인스턴스 B로 존재하며, 단순 수량 카운트가 아니다.
- 파츠 구매 시 인벤토리에 새 인스턴스가 추가된다.
- 공작실 조립 시 특정 인스턴스를 소비해 머신에 장착된다.
- 드랍 시스템(고난이도 행성 보상)도 동일하게 인스턴스를 추가한다.

### 파츠 옵션 시스템 (향후 설계 방향)

파밍(고난이도 행성 드랍)으로 획득하는 파츠는 **개별 옵션**을 가질 수 있다.

- 구매 파츠는 기본 스탯만 보유 (옵션 없음)
- 드랍 파츠는 인스턴스마다 고유한 옵션이 붙을 수 있다  
  예: `경량 프레임` (기본) vs `경량 프레임` (파견시간 +5% 옵션 보유)
- **티어는 고정 등급이 아닌 옵션 수와 수치값에 따라 결정**된다  
  예: 옵션 0개 → T1, 옵션 1개 + 낮은 수치 → T2, 옵션 2개 이상 → T3
- 인벤토리에서는 동일 종류라도 각 인스턴스를 개별 표기한다  
  예: `경량 프레임 / 경량 프레임 / 경량 프레임+` (수량 묶음 없음)

상점(PC 터미널 파츠 탭)에서는 **보유 수량을 표시하지 않는다.**  
파츠는 개별 인스턴스이므로 구매 버튼만 노출한다.  
보유 현황 확인은 공작실 인벤토리 탭에서 수행한다.

## 성장 구조

### 시설 업그레이드

- 수익 효율 향상
- 복귀 후 회전율 향상
- 파견 수용량 증가
- 지원 설비 해금

### 기지 확장

- 파견 슬롯 확장
- 운영 모듈 및 지원 공간 추가
- 기지 꾸미기 요소 확장

### 유닛 확장

- 파견 머신 추가 제작
- 파일럿 추가 고용
- 여러 자동 파견 동시 운용

### 플레이어 성장

- 직접 파견 성능 강화
- 클릭 효율 향상
- 임무 지속력 향상
- 수동 임무 보상 증가

## 확장 콘텐츠 방향

- 다양한 파견 지역 추가
- 지역별 환경 및 운용 조건 차별화
- 머신 스탯과 지역 적합도 간 상호작용 강화
- 파일럿 특성과 전문성 차별화
- 희귀 파츠, 특수 지역, 고수익 장기 임무 등 중후반 콘텐츠 추가

## 요약

플레이어가 직접 파견 임무를 수행하며 자금을 모으고, 그 수익으로 머신과 파일럿, 시설을 확장해 점점 더 큰 자동 파견 운영망을 구축해나가는 방치형/클리커 게임이다.
