# 파일럿 카드 뷰어 재설계 계획

작성일: 2026-05-28  
대상: `scripts/ui/shop_popup.gd`  
상태: 계획

---

## 배경 및 목표

현재 공고판은 3장 카드를 가로로 나열하는 방식이다.  
카드당 정보가 적고, 캐릭터가 아니라 스펙 시트처럼 보인다.

목표는 캐릭터를 "한 명씩 제대로 보는" 경험으로 전환하는 것이다.

- 카드 한 장이 화면을 지배 — 초상화 / 전신 / SD / 스탯 모두 노출
- 여러 장이 뒤에 살짝 겹쳐 보이는 덱 스타일
- 좌우 화살표로 한 장씩 넘기는 방식

---

## 카드 레이아웃

### 카드 크기: 320 × 190px (가로형 명함 비율)

```
┌─────────────────────────────────────────────────────────┐
│                                                         │
│  ┌──────────────┐  ┌────────────────────────────────┐  │
│  │              │  │  이름 (15px, bold)              │  │
│  │  전신 플레이스홀더  │  │  T1  ● ○ ○                  │  │
│  │  60 × 130 px │  │                                │  │
│  │  (컬러 rect)  │  │  임무 시간 -5%                 │  │
│  │              │  │  (보너스 설명)                  │  │
│  │  ┌──────┐   │  │                                │  │
│  │  │초상화 │   │  │  ─────────────────             │  │
│  │  │60×60 │   │  │  "첫 임무라도 긴장 안 해요"      │  │
│  │  │원형  │   │  │  (플레이버, 9px, italic 느낌)   │  │
│  │  └──────┘   │  │                                │  │
│  │  [SD 40×40]  │  │                                │  │
│  └──────────────┘  └────────────────────────────────┘  │
│                                                         │
│  ─────────────────────────────────────────────────      │
│  500 CR                            [고용하기]           │
└─────────────────────────────────────────────────────────┘
  좌측 아트 존 ~130px              우측 정보 존 나머지
```

### 좌측 아트 존 (130px 너비, 카드 전체 높이)

| 영역 | 크기 | 구현 |
|---|---|---|
| 전신 실루엣 | 60 × 110px | 파일럿 색 기반 `ColorRect` (어두운 배경 + 하이라이트 라인) |
| 초상화 | 56 × 56px (원형, 전신 위 하단 오버랩) | 기존 `_make_portrait()` 재사용 |
| SD | 40 × 40px (전신 아래) | 파일럿 색 기반 작은 원형, "SD" 글자 없이 픽셀 느낌 축소 |

실제 스프라이트 도입 시 `ColorRect`/원형을 `TextureRect`로 교체하면 된다.

### 우측 정보 존

- 이름: 15px, 볼드 계열, 밝은 흰색
- 티어 배지: `T1` 텍스트 + `●●○` 도트 3개 (기존 로직 유지)
- 보너스 설명: 11px, 보너스 타입별 색상 (기존 `_bonus_color()` 재사용)
- 구분선 1px
- 플레이버 텍스트: 9px, `"…"` 따옴표 포함, 파란 계열 희미한 색
- 스페이서 (나머지 공간)

### 하단 액션 존 (카드 하단 고정, 36px)

- 좌: 비용 `N CR` (크레딧 있으면 금색, 없으면 회색)
- 우: `고용하기` / `✓ ON BOARD` 버튼

## 덱 스타일 스택 효과

메인 카드 뒤로 나머지 2장이 살짝 비침 (실제 클릭 가능한 카드는 메인 1장만).

| 레이어 | 위치 오프셋 | 회전 | 불투명도 | z_index |
|---|---|---|---|---|
| 카드 0 (현재) | 기준 (0, 0) | 0° | 1.00 | 2 |
| 카드 1 (다음) | (+12, +5) | +4° | 0.60 | 1 |
| 카드 2 (그 다음) | (+22, +9) | +7° | 0.35 | 0 |

- 카드가 1장이면 스택 없음, 2장이면 뒤에 1장만 표시
- 고용 완료 파일럿이 공고판에 있으면 "ON BOARD" 상태로 스택에 포함

## 내비게이션

```
  ◀        [       카드       ]        ▶
            ●  ○  ○   (페이지 도트)
```

- `◀` / `▶` 버튼: 32×32px, 카드 좌우에 수직 중앙 정렬
- 공고판 파일럿이 0명이면 버튼 숨기고 안내 레이블 표시
- 페이지 도트: 현재 슬롯 인덱스 표시 (최대 3개), 카드 하단 중앙

### 넘기기 애니메이션

- 다음(▶): 새 카드가 우측 외부에서 0.15s ease-out으로 슬라이드인
- 이전(◀): 새 카드가 좌측 외부에서 0.15s ease-out으로 슬라이드인
- 스택 카드들도 위치 보간 (Tween 병렬)

## 팝업 레이아웃 변경

창 높이 300px 기준:

```
[헤더: 타이머 + 새로고침 + ✕]     26px
[오늘의 모집 파일럿 서브레이블]    14px
[카드 뷰어 영역: 내비 + 카드 + 도트]  EXPAND
[커스텀 파일럿 생성 버튼]          24px
```

카드 뷰어 영역 내부:
- 카드를 수평 중앙 배치
- 덱 스택이 카드 우측 하단으로 튀어나오는 형태 (overflow 허용)
- 내비 화살표는 카드 좌우 바깥에 float

## 구현 단계

### Step 1 — 새 카드 빌더
`_make_pilot_card_full(p: Dictionary) -> Control`  
- 320×190px PanelContainer
- 좌측 아트 존: 전신 ColorRect + 초상화 원형 + SD 원형
- 우측 정보 존: 이름 / 티어 / 보너스 / 플레이버
- 하단 액션 존: 비용 + 버튼

### Step 2 — 카드 뷰어 컨테이너
`_build_card_viewer() -> Control`  
- `_card_deck_root: Control` (overflow 허용 컨테이너)
- 덱 스택 카드 2장 (뒤 레이어, 터치 불가)
- 메인 카드 1장 (앞 레이어)
- `_page_dots: HBoxContainer` (하단 도트)
- `◀` / `▶` 버튼

### Step 3 — 상태 관리
- `_current_idx: int = 0`
- `_board_cards: Array` (현재 공고판 3장 캐시)
- `_rebuild_cards()` → `_board_cards` 갱신 후 뷰어 업데이트

### Step 4 — 넘기기 로직
`_go_to_card(new_idx: int, direction: int) -> void`  
- 방향(+1/-1)에 따라 새 카드 슬라이드인 Tween
- 스택 카드 위치/회전 보간

### Step 5 — 기존 카드 코드 제거
- `_card_row: HBoxContainer` 삭제
- `_make_pilot_card()` 삭제 (새 함수로 대체)

### Step 6 — 검증
- 파일럿 0명 (게임 초기)
- 파일럿 1명만 있을 때 (T2/T3 미해금 상태)
- 이미 고용된 파일럿 포함 공고판
- 유료 갱신 후 카드 뷰어 초기화 (idx → 0)
- 창 높이 300px 기준 클리핑 없이 카드 표시 확인
