# UI/UX 전면 재설계 계획

작성일: 2026-05-27  
상태: 확정 — 파노라마 함선 / 3구역 구조

---

## 확정 방향: 파노라마 함선 구조

기존 운영/제작 2패널 통합안은 패널 왕복을 줄이는 데 효과적이지만,
게임의 장기 목표인 `고난이도 행성 개척`, `파일럿 고용`, `함선 꾸미기`, `가구 설치`의 감각을 충분히 살리기 어렵다.

확정 방향은 플립으로 화면을 교체하는 방식이 아니라, 가로로 긴 함선 내부를 하나의 큰 캔버스로 만들고
선택한 구역으로 카메라가 이동하는 방식이다. 플레이어가 메뉴를 여는 것이 아니라
거대한 함선 안에서 브릿지/파일럿 라운지, 격납고, 관제실을 오가는 느낌을 목표로 한다.

### 확정 레이아웃

```text
[격납고 확장 구역] ← [격납고] ← [브릿지/파일럿 라운지] → [관제실]
```

- `브릿지/파일럿 라운지`: 중앙 홈. 파일럿 로밍, 생활 연출, 꾸미기 가구가 누적되는 기본 화면.
- `격납고`: 좌측 구역. 베이, 머신, 귀환 수령, 정비 상태를 담당. 베이가 늘어나면 좌측으로 계속 확장.
- `관제실`: 우측 구역. 항성지도 팝업 진입, 원정/행성 선택, 파견 명령, 상점/고용 진입을 담당.
- `항성지도 팝업`: 관제실에서 내려오는 큰 팝업. 가로로 긴 우주지도에서 아이콘형 행성들을 1행 간격의 가로 스크롤 그리드로 고르고, 행성 상세와 슬롯/기체 선택까지 이어지는 메인 편성 UI다.
- `조립 기능`: 별도 구역이 아니라 격납고 팝업에서 호출.
- `상점/고용 기능`: 관제실 내 팝업/오버레이로 호출.

### 이동 방식

- 네비게이션 탭: `격납고 / 브릿지 / 관제실` 3개를 기본으로 둔다.
- 탭 클릭: 해당 구역으로 카메라가 부드럽게 이동한다.
- 우클릭 드래그: 함선 내부를 직접 끌어서 이동한다.
- 위아래 이동: 당장은 쓰지 않지만, 나중에 숙소/생활층을 아래 데크로 추가할 수 있도록 구조적으로 열어둔다.

### 항성지도 팝업 상세 규칙

- 가로 스크롤 그리드는 아이콘 이미지를 중심으로 1행 간격이 약간 있는 형태로 구성한다.
- 마지막으로 선택한 행성의 위치를 기본값으로 유지한다.
- 행성 클릭 시 좌측 항성지도 영역은 15% 정도까지 축소된다.
- 우측 상세창에는 행성 상세정보와 이미지를 보여준다.
- 그 우측에는 해당 행성의 빈 슬롯과 정보 슬롯이 2행 그리드로 표시된다.
- 슬롯을 클릭하면 그 슬롯 우측에 기체 선택 팝업이 뜬다.
- 기체 선택 후에는 `파견 시작?` 확인 팝업을 작게 띄우고, 확인 시 출격을 시작한다.

### 격납고 베이 상세 팝업

- 베이 카드를 클릭하면 화면 전체 높이를 차지하는 전역 오버레이 팝업이 내려온다.
- 팝업 폭은 화면의 약 60%로 두고, 상단 중앙에 닫기 버튼을 둔다.
- 왼쪽에는 파일럿 초상화와 기본 상태를 배치한다.
- 가운데에는 몸통 / 무기 / 다리 순서의 RPG 장비칸 느낌의 스태거드 슬롯을 두고, 각 칸은 실제 파츠 스프라이트용 TextureRect 자리와 짧은 이름 캡션으로만 구성한다.
- 오른쪽 상단에는 파일럿 보너스를 포함한 머신 전체 스펙을 보여준다.
- 오른쪽 하단에는 파츠별 옵션 요약을 배치한다.
- 오른쪽에는 상태별 버튼 컬럼을 따로 두어 `조립 / 재조립 / 수리 / 관제실 이동 / 수령 / 해금`을 노출한다.
- 빈/도킹 슬롯 클릭 시 우측 슬라이드 드로어로 파일럿 목록 또는 파츠 인벤토리를 고른다.
- 도킹 상태는 재조립 버튼 없이 슬롯 클릭 교체로 처리하고, 수리 버튼은 예정 항목으로만 남긴다.
- 파견 중 Bay는 임무 정보, 도착 행성, 남은 시간만 보여주는 읽기 전용 요약으로 유지한다.
- 귀환 완료 Bay는 수령 버튼과 보상 요약만 보여준다.
- 우측 버튼 컬럼은 상태별 컨텍스트 액션만 두고, 중앙 장비칸은 읽기 전용 전시 영역으로 유지한다.
- 팝업은 위에서 아래로 스르르 내려오는 애니메이션을 사용한다.

### 참고 감각

- `Terraria`, `Oxygen Not Included`, `Ratopia`처럼 기능 공간이 하나의 긴 생활/운영 공간 안에 이어지는 느낌을 참고한다.
- 단, 이 프로젝트는 1920×300px 데스크톱 컴패니언 창이므로 전체 기지 시뮬레이션보다 짧은 이동, 명확한 상태등, 빠른 팝업 조작을 우선한다.

### 추천 우선순위와 작업 내용

| 우선순위 | 방향 | 작업 내용 |
|---|---|---|
| 1 | 플립 구조 폐기 | `PanelManager`의 전체 패널 교체 중심 구조를 장기적으로 카메라 이동/구역 포커스 구조로 전환한다. |
| 2 | 큰 함선 캔버스 + 카메라 이동 | 격납고, 브릿지/라운지, 관제실을 하나의 넓은 2D 공간에 배치하고 탭/드래그로 이동한다. |
| 3 | 브릿지/파일럿 라운지를 기본 홈으로 설정 | 게임 시작과 복귀의 기본 화면을 파일럿이 머무는 생활 공간으로 두어, 고용한 파일럿과 꾸민 가구가 항상 플레이 동기처럼 보이게 한다. |
| 4 | 관제/격납고 기능을 함선 내 구역으로 배치 | 격납고는 좌측 확장 구역, 관제실은 우측 구역으로 둔다. 관제실은 항성지도 팝업을 통해 원정/슬롯/기체 선택을 처리하고, 조립/상점/고용은 각 구역 팝업에서 호출한다. |
| 5 | 파일럿과 꾸미기가 항상 시각적으로 남는 구조 | 파일럿, 가구, 함선 확장 요소가 단순 메뉴 보상이 아니라 홈 공간에 누적되어 보이도록 한다. |

### 기존 2패널 통합안의 위치

아래의 운영/제작 2패널 통합안은 즉시 구현안이라기보다,
파노라마 함선 구조 안에서 `관제 구역`과 `제작 구역`의 기능 묶음을 설계할 때 참고한다.

## 배경 및 목표

### 현재 구조의 문제
- 1920×300px 형태임에도 패널이 전체를 *교체*하는 구조 → 이전 맥락 완전 소실
- 패널 5개(브릿지·격납고·파견관제·클리커·상점): 뎁스 최대 3단계
- 기능이 흩뿌려져 있어 "파견 내리기 → 수령"에 패널 이동 필요

### 설계 원칙
1. **함선 공간 탐색 방식으로 전환** — 플립 교체가 아니라 큰 함선 내부를 카메라가 이동
2. **3구역 기본 구조** — 좌측 격납고 / 중앙 브릿지·파일럿 라운지 / 우측 관제실
3. **기능은 구역 팝업으로 호출** — 조립은 격납고 팝업, 상점·고용은 관제실 팝업
4. **확장 방향을 공간으로 표현** — 베이는 좌측으로 확장, 숙소/생활층은 향후 아래 데크로 확장 가능
5. **글로벌 컨텍스트 항상 노출** — 크레딧·파견 현황은 카메라 위치와 무관하게 보임

### 우선순위 판단

- 이 계획은 `current-priorities.md`의 기존 "UI 재설계 완료" 이후에 진행하는 **상위 UX 구조 재설계**다.
- 현재 우선 검토 방향은 운영/제작 2패널 통합보다 큰 범위인 **파노라마 함선 구조**다.
- 기존 2패널 통합안은 기능 묶음과 이식 범위를 판단하는 참고안으로 유지한다.
- 사운드/SFX는 새 화면 구조가 확정된 뒤 붙인다. 제거되거나 이동할 UI에 사운드 이벤트를 먼저 심지 않기 위함이다.

## 이전 2패널 통합안 — 참고

```
현재 (4 서브패널)            →    변경 (2 서브패널)
────────────────────────────      ──────────────────────────
격납고    ┐                         관제 콘솔 클릭
파견 관제 ┘  →  [운영 패널]         → [운영 패널]
격납고 조립    ┐
PC 터미널 ┘  →  [제작 패널]         PC 터미널 클릭
                                    → [제작 패널]
```

| 이동 방식 | 목적지 | 흡수된 기능 |
|---|---|---|
| 브릿지 관제 콘솔 클릭 | 운영 패널 | 파견 관제 + 격납고 + 직접 파견 클리커 |
| 브릿지 PC 터미널 클릭 | 제작 패널 | 격납고 조립 팝업 + 관제실 상점(파츠·파일럿·함선강화) |

- 브릿지: 파일럿 로밍 공간 유지, 오브젝트 위치 이동 기능 유지
- 뎁스: 브릿지 ↔ 패널 2레벨 고정 (패널 내부는 슬라이드인으로 처리)
- 클리커: 독립 패널 제거 → 운영 패널 내부 상태로 흡수

## 슬라이드인 공통 규칙

- **속도**: 빠른 슬라이드 (0.18~0.22s, ease-out cubic)
- **방향**: 모든 패널에서 동일 방향 통일 (방향은 구현 시 확정)
- **닫기**: 역방향 슬라이드로 복귀
- anchor Tween 방식 유지 (현재 파견 관제·격납고 패턴 재사용)

## 브릿지 (변경 최소)

- 파일럿 로밍 애니메이션 유지
- 오브젝트 2개: 관제 콘솔, PC 터미널 (위치 드래그 가능)
- 하단 파견 현황 스트립: 파견중 N · 귀환완료 N (숫자만, 타이머 없음)
- 우측 유틸 바 (설정·사운드·최소화) 유지

## [운영] 패널 상세

파견 전 과정(출격→진행→수령)이 이 패널 하나에서 해결된다.

### 상태 1 — 기본: 행성 선택

행성을 고르는 단계이므로 행성 영역이 넓게, 베이는 우측에 미리보기 수준으로

```
┌───────────────────────────────────────┬─────────────┐
│  행성 선택 영역                         │  베이 그리드  │
│                                         │  (미리보기)  │
│  [섹터A ●] [섹터B] [섹터C]              │  [1][2][3]  │
│                                         │  [4][5]...  │
│  섹터A — 적 HP 2 · CR/킬 10 · 웨이브50 │             │
│                                         │             │
│  [직접 출격 ▶▶]  [파견 ▶]             │             │
└───────────────────────────────────────┴─────────────┘
         ~75% 폭                             ~25% 폭
```

### 상태 2 — 행성 선택 후: 베이 그리드 확장 (anchor Tween)

행성 선택 완료 → 행성 영역 축소, 베이 그리드가 메인으로 확장

```
┌──────────────┬──────────────────────────────────────┐
│  행성 compact │  베이 그리드 (16슬롯 가로 스크롤)      │
│              │                                        │
│  [A●][B][C]  │  [BAY1●][BAY2][BAY3][BAY4]...        │
│              │  (귀환완료 슬롯 펄스 표시)              │
│  [직접출격]  │                                        │
│  [파견▶]    │                                        │
└──────────────┴──────────────────────────────────────┘
    ~25% 폭                  ~75% 폭
```

취소 버튼(또는 행성 영역 클릭) → 상태 1로 역방향 tween 복귀

### 상태 3 — 파견 모드 (파견▶ 클릭)

기존 파견 관제의 anchor Tween 2-state 그대로 재사용  
planet_zone compact + slot_zone 슬라이드인

### 상태 4 — 베이 상세 (베이 카드 클릭)

기존 격납고의 좌측 슬라이드인 그대로 재사용  
상세 패널 30% + 베이 그리드 70%

수령 플로우: 베이 그리드에서 귀환완료 슬롯 클릭 → 상세 패널 → [수령▶]  
(브릿지→격납고 이동 불필요)

### 상태 5 — 직접 출격 모드 (직접출격▶▶ 클릭)

클리커 전투 영역이 패널 전체를 덮으며 슬라이드인

```
┌───────────────────────────────────────────────────────┐
│  섹터A  처치: 12/50  보류: 120 CR          [복귀]      │
├───────────────────────────────────────────────────────┤
│                                                         │
│    👾        👾      👾          👾       👾            │
│         👾                  👾                          │
│                                                         │
└───────────────────────────────────────────────────────┘
```

복귀 → 슬라이드아웃, 상태 2(베이 그리드 확장 상태)로 복귀  
수익 수령은 복귀와 동시에 자동 처리

## [제작] 패널 상세

파츠 구매 → 인벤토리 확인 → 조립이 이 패널 하나에서 해결된다.

### 레이아웃 — 좌측 카테고리 메뉴 + 우측 콘텐츠

```
┌──────────────┬────────────────────────────────────────┐
│  파츠 구매   │                                          │
│  ──────────  │   선택한 카테고리 콘텐츠                  │
│  조립        │   (현재 PC 터미널 우측 콘텐츠 구조 유지)  │
│  파일럿      │                                          │
│  함선 강화   │                                          │
│  ──────────  │                                          │
│  (시설)     │                                          │
│  (꾸밈)     │                                          │
└──────────────┴────────────────────────────────────────┘
   ~180px               나머지 전체
```

| 카테고리 | 흡수 출처 | 내용 |
|---|---|---|
| 파츠 구매 | 관제실 파츠 구매 팝업 | 몸체/무기/다리 인벤토리형 구매 슬롯 |

## 피드백 (2026-05-27)

### 운영 패널 — 베이 카드 레이아웃 스케일 문제
베이 수가 16개, 32개로 늘어날 경우 가로 카드 1행 레이아웃으로는 대응 불가.
슬롯 수가 많아져도 한눈에 파악할 수 있는 레이아웃 방식 재검토 필요.

### 운영 패널 — 진행 순서 수정
현재 와이어프레임은 베이 현황을 기본 진입 상태로 잡았으나,
실제 플레이 순서는 **행성 선택 → 베이 선택** 이 맞음.
나중에는 여러 베이를 동시 선택해 한꺼번에 파견하는 흐름도 고려.
→ 운영 패널 기본 진입 상태를 행성 선택으로 변경 검토.

### 직접 파견 — 플레이어 전용 베이
임무 결과 화면을 별도 씬으로 만드는 것보다,
직접 파견도 "플레이어 머신의 임무"로 취급해 **플레이어 전용 베이(BAY 0)** 를 하나 두는 방향 검토.
귀환 후 베이에 플레이어 머신이 대기 중인 모습으로 표현 → 별도 결과 화면 불필요.
자동 파견 베이와 동일한 수령 UX 흐름으로 통합 가능.

### 제작 패널 — 파츠 구매 UI
현재 리스트 행 나열 방식은 밀도가 높고 게임답지 않음.
다른 표현 방식 검토 필요 (예: 카드형, 도감형, 장비창형 등).

### 제작 패널 — 조립 UI
현재 5열 스프레드시트 형태는 게임 느낌 부족.
더 게임스러운 조립 인터페이스 방향 검토 필요
(예: 머신 실루엣에 파츠를 붙이는 방식, 슬롯 드래그 등).
