# 자동 파견 크레딧 스케일 작업계획서 (초안)

작성일: 2026-05-31

## 문제 정의 (데이터 실측)

자동 파견 1사이클 정산 (`dispatch_manager._start_returning`):
`raw_credits = 무기 CR/s × 파견시간(몸체 티어 값)`

| 항목 | 값 |
|---|---|
| 무기 CR/s | T1=2 / T2=5 / **T3=12 (상한)** |
| 몸체 파견시간 | T1=30 / T2=75 / **T3=150s** |
| T3 풀스펙 raw | 12 × 150 = **1,800 CR (고정 상한)** |
| 행성 CP 보상 | sector_a ≈100 → sector_h ≈200, **sector_i~t는 후반 곡선으로 상향** |
| 1사이클 최대 | `raw_credits + 행성 CP + 재료 보상`

대비:
| 항목 | 스케일 |
|---|---|
| 직접 클리커 `credit_per_kill` | 10(a) → 24,000(t) = **2,400배** |
| 다음 행성 해금 비용 | 50(b) → 4×10¹¹(t), 단계당 **×3.3~3.8** |

**핵심 모순**: 초반과 후반이 같은 CP 곡선을 쓰면 자동 파견이 끝까지 평평해져서, 후반 해금과 자원 투자에서 존재감이 사라진다.  
해결은 `sector_i~sector_t` 구간만 CP를 후반형으로 재곡선화하고, 자동 파견의 정체성을 `재료 주가치 + 보조 CP`로 고정하는 것이다.

## 방향 (확정: ②축 + ① 결합)

정체성: **직접 클리커 = CP 엔진 / 자동 파견 = 재료 엔진 + 보조 CP**

### 변경 항목
1. **행성 CP 재산정 (①)**: `data/planet_data.gd`의 `guaranteed_rewards.cp`를 `credit_per_kill` 비례 곡선으로
   재산정. `sector_i~sector_t`만 후반 보정하고, 초반 섹터는 유지한다.
2. **재료 = 자동 파견 주가치 (②)**: 시설/구역/기지 확장 비용을 재료 중심으로 유지·강화(기존 다중 재화 구조 활용),
   UI 카피/안내를 "자동=재료, 클리커=CP"로 정리.
3. **해금 비용 곡선 점검**: ×3.5 일정(cliff 없음) 유지하되 후반 CP 공급(1)과 균형 재계산.
4. **무기 T4+ (③)**: 2차 과제로 분리 — 이번 범위 제외.
