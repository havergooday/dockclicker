# Living Ship Design

## 목적

이 문서는 `Dispatch Clicker`의 생활형 함선 기준선을 정리한다.  
기존의 `파견 중심 방치형/클리커 RPG` 구조를 유지하되, 게임의 중심 재미를 `파츠 파밍`이 아니라 `비어 있던 함선을 파일럿들과 함께 가꾸고 살아가는 경험`으로 옮긴다.

핵심 장르 정의는 아래와 같다.

```text
작은 함선을 가꾸고 파일럿들과 함께 살아가는 캐릭터 운영 방치 게임
```

파견은 여전히 주요 자원 수급 수단이다. 다만 최종 목적은 더 높은 숫자를 만드는 것이 아니라, 함선에 시설을 추가하고, 파일럿들이 생활하며, 공간이 점점 복작복작해지는 변화를 만드는 것이다.

## 현재 구현 상태

2026-05-30 기준으로 아래 요소가 실제 코드에 반영되어 있으며, 현재 기준선으로 유지한다.

- `ShipCanvas` 파노라마 함선 캔버스
- 관제실 `항성지도` / `파일럿 고용` / `파츠 구매` / `시설관리` 팝업
- `HangarZone` 전역 베이 상세 팝업과 격납고 해금
- `GameState` 다중 재화(`cp / alloy / supplies / circuit`)
- 파일럿 생활 상태(`fatigue / stress / mood`)
- 라운지 시설(`소파 / 게임기 / 커피 머신`)
- 숙소 침대 회복 틱

## 핵심 문제 인식

단순한 방치형 RPG 루프는 금방 반복 피로가 온다.

```text
파견 → 보상 → 파츠 강화 → 더 좋은 파견
```

이 구조만으로는 유사한 무료 방치형 RPG, 파밍 RPG와 직접 경쟁하게 된다. 아이템 파밍, 도트 그래픽, 거래 시스템이 잘 갖춰져 있어도 플레이어가 `다음에 뭘 보고 싶은지`가 약하면 재미가 오래가지 않는다.

따라서 이 프로젝트의 차별점은 `고품질 파밍 RPG`가 아니라, `함선 생활 관찰`, `파일럿 애착`, `공간 성장`이어야 한다.

## 새 코어 판타지

처음의 함선은 삭막하고 비어 있다.

- 불 꺼진 브릿지
- 낡은 격납고 1칸
- 앉을 곳 없는 라운지
- 파일럿 없음
- 생활 시설 없음
- 플레이어 혼자 직접 파견을 나감

플레이어는 직접 파견과 초기 운영으로 자원을 모아 함선을 조금씩 복구한다. 첫 소파, 첫 식탁, 첫 파일럿, 첫 자동 파견, 첫 개인 공간이 생기면서 함선은 단순 메뉴 화면이 아니라 `사람 사는 곳`으로 바뀐다.

## 새 코어 루프

기존 루프는 아래와 같이 재해석한다.

```text
파견 → 자원 획득 → 시설/공간 추가 → 파일럿 생활 변화 → 운영 판단 → 다음 파견
```

플레이어가 반복해서 확인하고 싶어야 하는 대상은 `보상 숫자`만이 아니다.

- 지금 파일럿들이 어디서 무엇을 하는지
- 누가 피곤하거나 스트레스를 받았는지
- 새 시설 때문에 행동 패턴이 어떻게 바뀌었는지
- 어떤 재료가 부족해서 다음에 어디로 보낼지
- 특정 파일럿을 쉬게 할지, 무리해서 한 번 더 보낼지

## 림월드식 감성의 적용 범위

목표는 복잡한 생존 시뮬레이션을 만드는 것이 아니다.  
림월드에서 가져올 감성은 `옹기종기 모여 사는 캐릭터 관찰`, `작은 사건이 상태 변화로 남는 느낌`, `공간이 생활 기능을 갖는 구조`다.

적용할 것:

- 파일럿이 함선 안을 돌아다닌다.
- 피곤하면 쉬고, 배고프면 식당으로 가고, 다치면 의무실을 찾는다.
- 친한 파일럿끼리 가까이 앉거나 대화한다.
- 싫어하는 시설, 선호하는 시설, 선호 행성이 있다.
- 파견 결과가 피로, 스트레스, 부상, 숙련도, 기분으로 남는다.

초기에는 복잡한 욕구 시뮬레이션 대신, 몇 개의 상태값과 행동 규칙만 사용한다.

## 파일럿 생활 시스템

파일럿은 단순한 보너스 수치가 아니라 함선 생활의 주체다.

### 기본 상태

- `피로`: 파견 후 증가, 휴식 시설에서 감소
- `스트레스`: 위험 파견, 연속 파견, 불편한 환경에서 증가
- `기분`: 좋은 시설, 선호 행동, 관계 이벤트로 변화
- `부상`: 위험 파견 결과로 발생, 의무실 또는 의료 키트로 회복
- `숙련도`: 특정 행성 또는 임무 타입 반복으로 증가

### 성격과 취향

파일럿은 간단한 태그를 가진다.

- 활발함: 훈련 매트, 게임 콘솔 선호
- 차분함: 책장, 화분, 창가 좌석 선호
- 사교적: 식탁, 소파, 공동 활동 선호
- 독립적: 개인 보관함, 개인실, 조용한 공간 선호

이 태그는 긴 스토리보다 행동 선택과 짧은 반응에 우선 사용한다.

### 작업 내용

- 파일럿 상태값 데이터 모델 추가
- 상태값에 따른 행동 선택 로직 추가
- 성격/취향 태그 데이터 추가
- 파견 결과가 파일럿 상태에 영향을 주도록 정산 로직 확장
- 브릿지/라운지/숙소 로밍이 상태 기반 행동을 반영하도록 개선

## 라운지와 생활 시설

라운지는 단순 빈 공간이 아니라 함선 생활의 중심이 된다.  
처음에는 삭막한 빈 구역이고, 플레이어가 시설물을 설치하면서 행동과 효과가 열린다.

### 시설 설계 원칙

시설물은 단순 수치 버프가 아니라 `행동을 여는 장치`여야 한다.

나쁜 예:

```text
소파: 피로 회복 +5%
```

좋은 예:

```text
소파:
- 휴식 행동 해금
- 앉은 파일럿은 피로 회복
- 친한 파일럿이 옆자리에 앉을 확률 증가
- 라운지 분위기 +1
```

### 초기 시설 예시

| 시설물 | 주요 효과 | 화면 변화 |
|---|---|---|
| 소파 | 휴식 행동 해금, 피로 회복 | 파일럿이 앉거나 눕는다 |
| 식탁 | 식사 행동 해금, 대화 확률 증가 | 여러 명이 같이 앉는다 |
| 간이 주방 | 생활물자 소비로 식사 버프 | 조리/식사 행동이 생긴다 |
| 게임 콘솔 | 스트레스 감소, 관계 증가 | 둘이 같이 게임한다 |
| 책장 | 차분한 파일럿 기분 회복 | 혼자 책을 읽는다 |
| 화분 | 라운지 분위기 증가 | 주변에서 쉬는 빈도가 오른다 |
| 게시판 | 파견 목표 표시, 자동 파견 효율 보조 | 파일럿이 확인하러 간다 |
| 훈련 매트 | 숙련도 성장 보너스 | 운동 행동이 생긴다 |
| 커피 머신 | 파견 준비 시간 감소 | 출격 전 커피를 마신다 |
| 의료 키트 | 경상 회복 | 다친 파일럿이 사용한다 |

### 슬롯 방식

초기 구현은 자유 배치가 아니라 고정 슬롯 방식으로 시작한다.

```text
벽면 슬롯: 게시판 / 책장 / 장식
휴식 슬롯: 소파 / 벤치 / 안락의자
테이블 슬롯: 식탁 / 작업대 / 게임 테이블
서비스 슬롯: 커피 머신 / 간이 주방 / 의료 키트
장식 슬롯: 화분 / 조명 / 포스터
```

자유 배치는 이후 확장으로 남긴다.

### 작업 내용

- 라운지 시설 슬롯 데이터 구조 추가
- 시설물 데이터 테이블 추가
- 설치/교체 UI 추가
- 시설별 행동 포인트 정의
- 파일럿 로밍이 사용 가능한 시설물을 목적지로 선택하도록 개선
- 시설 효과를 피로, 스트레스, 기분, 관계, 파견 준비 시간에 연결

## 함선 공간 성장

함선은 처음부터 완성된 공간이 아니라 단계적으로 복구되고 확장된다.

### 초기 상태

- 브릿지: 최소 기능만 작동
- 격납고: 기본 베이 일부만 사용 가능
- 라운지: 빈 공간
- 숙소: 잠김 또는 임시 구역
- 식당/의무실/정비실/온실: 미개방

### 확장 방향

- 라운지 확장: 더 많은 파일럿이 동시에 쉼
- 식당 추가: 식사 행동과 단체 대화 해금
- 의무실 추가: 부상 회복, 위험 파견 부담 감소
- 정비실 추가: 머신 수리/효율 관리
- 숙소 추가: 파일럿 개인 회복과 개인화
- 온실 추가: 생활물자 일부 자급 또는 기분 보너스

### 작업 내용

- 함선 구역 해금 상태 저장
- 구역별 시설 슬롯 수 정의
- 구역 해금 비용 추가
- 잠긴 구역의 시각 상태와 해금 UI 구현
- 구역 해금 시 파일럿 행동 후보 확장

## 다중 재화와 특수재료

재화는 `CP` 하나만 두지 않는다.  
CP는 범용 운영비로 유지하고, 특수재료는 시설, 구역, 생활 콘텐츠 해금에 사용한다.

### 기본 자원 구조

| 자원 | 역할 | 주요 획득처 | 주요 사용처 |
|---|---|---|---|
| CP | 기본 화폐 | 모든 파견 | 고용, 기본 구매, 유지비 |
| 합금판 | 구조 확장 재료 | 폐기 위성, 광산 | 방 증축, 격납고 확장 |
| 회로칩 | 전자/관제 재료 | 도시 잔해, 기계 행성 | 관제실, 자동화, 터미널 |
| 생체샘플 | 의료/연구 재료 | 생태 행성 | 의무실, 회복 시설 |
| 에너지코어 | 고급 전력 재료 | 위험 지역, 성운 | 전력실, 고급 설비 |
| 생활물자 | 생활 시설 재료 | 교역 항로 | 식당, 라운지, 가구 |
| 기억파편 | 캐릭터 콘텐츠 재료 | 특수 파견, 관계 이벤트 | 개인 이벤트, 개인실, 특성 개방 |

### 드랍 원칙

특수재료는 확률 드랍만으로 두지 않는다.  
지역별 보장 수급을 두고, 희귀 보상만 확률화한다.

```text
폐기 위성 5분 파견:
- CP 80~120
- 합금판 2~4 보장
- 회로칩 20% 확률

교역 항로 8분 파견:
- CP 60~90
- 생활물자 3~5 보장
- 파일럿 기분 회복 가능
```

### 작업 내용

- GameState 재화 구조를 CP 단일에서 다중 자원으로 확장
- 저장 데이터 마이그레이션 추가
- 파견 지역별 보장 재료/확률 재료 테이블 추가
- 보상 UI에 다중 자원 표시 추가
- 시설/구역/고용 비용이 다중 자원을 요구하도록 확장

## 파견 지역의 역할 재정의

행성은 단순 난이도/수익 차이가 아니라 필요한 재료를 얻으러 가는 장소가 된다.

| 지역 타입 | 주요 보상 | 플레이 목적 |
|---|---|---|
| 폐기 위성 | 합금판, CP | 초반 구조 확장 |
| 버려진 도시 | 회로칩, 합금판 | 자동화/관제 강화 |
| 생태 행성 | 생체샘플, 생활물자 | 의료/회복/온실 |
| 교역 항로 | 생활물자, CP | 라운지/식당 개선 |
| 불안정 성운 | 에너지코어, 희귀 파츠 | 고급 설비 해금 |
| 기억 잔해 | 기억파편 | 파일럿 개인 콘텐츠 |

### 작업 내용

- 기존 행성 데이터에 지역 타입과 주요 재료 필드 추가
- 항성지도에서 주요 보상 재료를 명확히 표시
- 잠긴 지역도 필요한 해금 목표와 대표 보상을 미리 보여주기
- 파일럿 전문 행성/선호 행성과 연결

## 관계와 애착

연애 시뮬레이션의 장점을 일부 가져오되, 정통 비주얼노벨 구조로 가지 않는다.  
목표는 긴 스토리 루트가 아니라 `캐릭터에게 애착이 생기는 운영`이다.

### 적용할 요소

- 파일럿별 신뢰도
- 파일럿끼리 친밀도/갈등
- 선호 시설, 선호 행성, 싫어하는 임무
- 짧은 말풍선 반응
- 개인실/개인 장식/기억파편을 통한 개인 콘텐츠 해금

### 피할 요소

- 긴 대화 이벤트 중심 구조
- 루트별 엔딩 중심 구조
- 반복 선물 노가다
- 대량의 스토리 텍스트 의존

### 작업 내용

- 파일럿 신뢰도/관계 데이터 추가
- 짧은 말풍선 시스템 추가
- 시설 이용, 파견 결과, 휴식 행동이 관계에 영향을 주도록 연결
- 기억파편을 사용하는 개인 콘텐츠 해금 구조 설계

## MVP 적용 순서

피벗을 한 번에 구현하지 않고 작은 단위로 검증한다.

1. 다중 재화 추가
   - CP 외 `합금판`, `생활물자`, `회로칩`부터 시작
   - 기존 행성 보상에 보장 재료를 붙인다.

2. 라운지 시설 슬롯 추가
   - 고정 슬롯 5개
   - 설치 가능한 시설물 6~10종
   - 시설 설치 비용은 다중 재화를 사용한다.

3. 파일럿 상태값 추가
   - 피로, 스트레스, 기분부터 시작
   - 파견 후 상태 변화와 대기 중 회복을 연결한다.

4. 시설 기반 행동 추가
   - 소파 휴식
   - 식탁 식사
   - 게임 콘솔 놀이
   - 의료 키트 회복

5. 함선 구역 해금 추가
   - 식당 또는 의무실 중 하나를 먼저 해금 대상으로 구현한다.

6. 관계/말풍선 추가
   - 긴 이벤트 대신 짧은 상태 반응을 우선 구현한다.

## 성공 기준

이 방향의 성공 기준은 아래 질문에 답할 수 있는지다.

- 플레이어가 보상 숫자뿐 아니라 함선 내부 변화를 보고 싶은가?
- 새 시설을 설치했을 때 파일럿 행동이 실제로 달라지는가?
- 파견 지역 선택이 `돈 효율` 외의 이유를 가지는가?
- 파일럿을 무한 반복 출격시키는 것보다 쉬게 하거나 돌보는 판단이 의미 있는가?
- 처음의 삭막한 함선과 1~2시간 뒤 함선의 분위기가 눈에 띄게 다른가?

이 기준을 만족하면 게임은 `파견 방치 RPG`보다 `작은 함선 생활 운영 게임`에 가까워진다.
