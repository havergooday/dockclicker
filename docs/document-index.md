# 문서 인덱스

## 개요

이 문서는 `Dispatch Clicker` 프로젝트의 문서 허브다.  
프로젝트를 이해하거나 작업을 이어갈 때, 이 문서를 시작점으로 삼아 필요한 문서로 이동할 수 있도록 구성한다.

## 우선 읽을 문서

1. [START HERE](D:/Project/side/dockclicker/docs/start-here.md)
프로젝트를 새로 로드했을 때 가장 먼저 확인하는 진입 문서

2. [프로젝트 정보](D:/Project/side/dockclicker/docs/project-information.md)
프로젝트 목표, 기술 스택, 현재 구조, 검증 방식 등 프로젝트 전반을 빠르게 파악할 때 본다.

3. [구현된 기능](D:/Project/side/dockclicker/docs/implemented-features.md)
현재 저장소에 실제로 반영된 기능, 구조, 문서화 범위를 확인할 때 본다.

4. [아직 없는 기능](D:/Project/side/dockclicker/docs/missing-features.md)
아직 구현되지 않았고 앞으로 만들어야 할 핵심 기능을 확인할 때 본다.

5. [AI 작업 가이드](D:/Project/side/dockclicker/docs/ai-workflow.md)
AI가 어떤 순서로 문서를 읽고, 어떤 기준으로 수정해야 하는지 확인할 때 본다.

6. [현재 우선순위](D:/Project/side/dockclicker/docs/current-priorities.md)
지금 시점에서 무엇부터 구현해야 하는지 빠르게 확인할 때 본다.

## 문서 탐색 가이드

- 게임 아이디어와 시스템을 이해하고 싶으면 `기획서`부터 본다.
- 화면 구조와 비주얼 방향을 알고 싶으면 `디자인 문서`를 본다.
- 현재 프로젝트에 실제로 무엇이 있는지 알고 싶으면 `프로젝트 정보`와 `구현된 기능`을 본다.
- 다음 작업 우선순위를 잡고 싶으면 `아직 없는 기능`을 본다.
- AI가 바로 작업에 들어가려면 `START HERE` 다음에 `AI 작업 가이드`와 `현재 우선순위`를 함께 본다.
- 이미 확정된 기술 선택을 확인하려면 `기술 결정 기록`을 본다.
- 커밋 규칙을 확인하려면 `커밋 워크플로우`를 본다.
- 특정 시스템을 깊게 작업하려면 `시스템 문서 인덱스` 아래 문서를 본다.
- 코드와 폴더 책임을 확인하고 싶으면 `Godot 프로젝트 구조`를 본다.
- 구현 작업의 시작 지점을 잡고 싶으면 `구현 계획 문서`를 본다.
