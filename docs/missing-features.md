# 아직 없는 기능

## 개요

아직 구현되지 않았지만 현재 기획 기준으로 만들어야 할 기능을 정리한다.

## 핵심 UI 시스템

- **UI/UX 통합 재설계** — 파노라마 함선 캔버스 진행 중 (`docs/plans/2026-05-27-ux-redesign.md`)
  - 완료: `ShipCanvas` 메인 루트, 관제실 `항성지도` 팝업, `HangarZone` 베이 그리드 + 오버레이 팝업, `ShopPopup` 파일럿 공고판, `PartsShopPopup` 파츠 구매 팝업, 브릿지 파일럿 로밍, 격납고 조립 팝업
  - 남은 작업: 격납고 좌측 확장, 관제실/격납고 UI 미세 조정
- SD 캐릭터 idle 애니메이션 — 현재 컬러 사각형 플레이스홀더, 픽셀 아트 스프라이트 미구현
- 상태 표시줄 위치 옵션 (상/하/좌/우, 설정에서 변경) — 기획 완료, 미구현

## 직접 파견 클리커 (추가 개선)

- 행성별 고유 외계생물 픽셀 아트 (현재 6종 글리프/이동 패턴 + 색상 구분까지 구현, 스프라이트 교체 예정)
- 플레이어 기체 무기 종류 변경 (클릭 범위·콤보·자동공격은 구현 완료)

## 파츠 옵션 시스템 (남은 작업)

- 티어를 고정 등급이 아닌 옵션 수·수치값 기반으로 동적 결정 (현재는 tier + 옵션 2개 병행)
- 설계 원칙은 `docs/dispatch-clicker-design.md` 파츠 옵션 시스템 섹션 참조
- 참고: 드랍 파츠 옵션 부착 / 효과 적용 / 파츠 분해 환급은 구현 완료
- 참고: 공작실·베이 팝업 모두 iid 기반 인스턴스 단위 선택 구현 완료 (동일 티어 옵션 구분)

## 자동 파견 — 미구현 기능

- 자동 재파견 다중 사이클 오프라인 계산 (현재: 오프라인 중 1사이클만 계산)
- 전원 출격 버튼
- 자동 파견 수익 상세 내역 표시

## 격납고 — 미구현 비주얼

- **메크 스프라이트 조합 시스템**: 동체/무기/하체 tier 조합에 따라 완성된 스프라이트 하나를 렌더링
  - 각 파츠별 tier 스프라이트를 레이어 합성하거나 단일 조합 스프라이트셋으로 구성 예정
  - 현재 베이 상세 팝업의 머신 실루엣 영역에 장착된 파츠 스프라이트를 표시하는 방식으로 연결 예정
  - 구현 시 `hangar_bay_popup.gd`의 파츠 슬롯 렌더링 부분을 교체하면 됨

## 자동 파견 모니터링 (MVP 이후)

- CCTV 소형 창 — 저해상도/노이즈 필터 느낌의 작업 현황 확인
- 구체적 표시 방식(상태바, 픽셀 뷰, 텍스트 피드)은 추후 결정

## 성장 시스템

- 시설 업그레이드
- 기지 확장
- 파견 효율 상승

## 파일럿 시스템 — 미구현 기능

- 파일럿 커스텀 복장 그래픽 (현재는 이름 이니셜 원형만 표시)
- 브릿지 파일럿 idle/픽셀 스프라이트 (현재 단색 원형 플레이스홀더)
- 파일럿 성장 시스템 — 임무 경험치, 티어 승급 (향후)
- 유전자 조합 / 파일럿 특성 전이 (중후반 콘텐츠 예정)

## 사운드

- BGM 트랙 및 AudioServer BGM/SFX 버스 설정 (버스 추가 시 사운드 패널에 자동 표시됨)
- SFX — 클릭음, 수령음, 파견 출발음 등
