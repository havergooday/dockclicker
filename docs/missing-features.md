# 아직 없는 기능

## 개요

아직 구현되지 않았지만 현재 기획 기준으로 만들어야 할 기능을 정리한다.

## 핵심 UI 시스템

- **UI/UX 통합 재설계** — 파노라마 함선 캔버스 진행 중 (`docs/plans/2026-05-27-ux-redesign.md`)
  - 완료: `ShipCanvas` 메인 루트, 관제실 `항성지도` 팝업, `HangarZone` 베이 그리드
  - **베이 상세 팝업 UX 미확정**: 현재 슬라이드인 패널 임시 구현, 재설계 논의 필요
  - 남은 작업: 격납고 좌측 확장, 격납고 팝업(공작실 이식), 브릿지/파일럿 라운지 홈 연출, 관제실 상점/고용 팝업
- SD 캐릭터 idle 애니메이션 — 현재 컬러 사각형 플레이스홀더, 픽셀 아트 스프라이트 미구현
- 상태 표시줄 위치 옵션 (상/하/좌/우, 설정에서 변경) — 기획 완료, 미구현

## 직접 파견 클리커 (추가 개선)

- 행성별 고유 외계생물 그래픽/종류 (현재 👾 단일)
- 플레이어 기체 업그레이드 추가 — 클릭 범위, 무기 종류
- 고난이도 행성 머신 파츠 드랍

## 파츠 옵션 시스템 (향후)

- 드랍 파츠에 개별 옵션 부여 (구매 파츠는 옵션 없음)
- 티어는 고정 등급이 아닌 옵션 수·수치값 기반으로 결정
- 인벤토리에서 동일 종류 파츠도 개별 표기 (수량 묶음 없음)  
  예: `경량 프레임 / 경량 프레임 / 경량 프레임+`
- 설계 원칙은 `docs/dispatch-clicker-design.md` 파츠 옵션 시스템 섹션 참조

### 조립 탭 iid 기반 선택 리팩터링 (옵션 시스템 구현 시 필수)

현재 조립 탭(`panel_workshop.gd`)의 파츠 선택은 `_equipped = {type: tier}` 구조로
tier 번호 기반이다. 기능상 동작하지만 옵션 시스템 도입 시 동일 종류·티어 내
특정 인스턴스를 구별할 수 없다.

- `_equipped` 를 `{type: iid}` 구조로 전환
- `_make_part_row()` 가 `iid` 를 받아 선택/장착 처리
- `assemble_machine()` 에 tier 대신 iid 전달 → `consume_part_by_iid()` 호출

## 자동 파견 — 미구현 기능

- 자동 재파견 다중 사이클 오프라인 계산 (현재: 오프라인 중 1사이클만 계산)
- 전원 출격 버튼
- 자동 파견 수익 상세 내역 표시

## 격납고 — 미구현 비주얼

- **메크 스프라이트 조합 시스템**: 동체/무기/하체 tier 조합에 따라 완성된 스프라이트 하나를 렌더링
  - 각 파츠별 tier 스프라이트를 레이어 합성하거나 단일 조합 스프라이트셋으로 구성 예정
  - 현재 베이 카드 내 플레이스홀더(회색 영역) 위치에 표시됨
  - 구현 시 `panel_hangar.gd`의 `_make_bay_card()` 내 `sprite_ph` Panel 부분을 교체하면 됨

## 자동 파견 모니터링 (MVP 이후)

- CCTV 소형 창 — 저해상도/노이즈 필터 느낌의 작업 현황 확인
- 구체적 표시 방식(상태바, 픽셀 뷰, 텍스트 피드)은 추후 결정

## 성장 시스템

- 시설 업그레이드
- 기지 확장
- 파견 효율 상승

## 파일럿 시스템 — 미구현 기능

- 파일럿 커스텀 복장 그래픽 (현재는 이름 이니셜 원형만 표시)
- 브릿지 idle 애니메이션 (파일럿이 브릿지를 돌아다니는 스프라이트)

## 사운드

- BGM 트랙 및 AudioServer BGM/SFX 버스 설정 (버스 추가 시 사운드 패널에 자동 표시됨)
- SFX — 클릭음, 수령음, 파견 출발음 등

## PC 터미널 — 후속 폴리싱 미완료

메뉴형 구조 자체는 구현 완료. 다만 아래 확장/폴리싱은 아직 남아 있다.

### 향후 카테고리
- `시설` 카테고리 실기능
- `꾸밈` 카테고리 실기능

### 파일럿 연출 보강
- 커스텀 생성 미리보기의 전용 일러스트/픽셀 스프라이트
- 파일럿 상세 패널의 추가 성격/로그/대사 연출

### 함선 강화 확장
- 클릭 범위 강화
- 무기 교체 또는 직접 파견 무기 계열 확장
- ~~자동 공격 업그레이드~~ ✓ 구현 완료
