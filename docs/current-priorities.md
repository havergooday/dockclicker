# 현재 우선순위

## 현재 단계

직접 파견 + 자동 파견 기본 루프 모두 완성.  
파츠 개별 아이템 시스템 리팩터링(v3) + 공작실 인벤토리 탭 구현 완료.  
PC 터미널 메뉴형 레이아웃 구현 완료.  
ShipCanvas (가로 파노라마 함선 캔버스) 구현 완료.  
항성지도 팝업 전면 재설계 완료 — 행성 슬롯 4행 그리드, 베이 팝업 슬라이드인, 우클릭 드래그 스크롤.  
직접 파견 클리커 UX 개선 완료 — "미션완료" / "함선복귀" 플로팅 텍스트.  
격납고 구역 (`HangarZone`) 구현 완료 — 베이 그리드 우측 정렬, 상태별 상세 패널, 귀환 수령, 구역 격벽.

현재 작업은 **UI/UX 통합 재설계** Phase 2~3 진행 중.  
격납고 구역화 1차 완료. 베이 상세 팝업 UX 방향 논의 후 이어서 진행 예정.  
사운드 시스템은 이 구조 통합 이후에 붙인다.

## 논의 미완료 항목

- **베이 상세 팝업 UX** ← 다음 논의 필요  
  현재 베이 클릭 시 좌측 슬라이드인 패널(30%)로 구현되어 있으나, 표시할 정보와 액션의 구성 방향을 재검토해야 함.  
  머신 상태 표시, 파일럿 배정, 파견/수령 액션을 어떤 형태(슬라이드인 패널 / 오버레이 팝업 / 인라인 확장 등)로 구성할지 확정 필요.
- 관제실 UI/UX: 항성지도 팝업 구현 완료. 상점/고용 팝업, 직접 출격 진입 흐름의 정보 우선순위 미확정
- 격납고 UI/UX: 베이 확장, 귀환 수령, 머신 관리, 공작실 팝업을 어떤 흐름으로 구성할지 확정
- 반복 조작 대응: 쉬리릭 이동 연출과 빠른 조작 편의 사이의 균형 확정
- 아래 데크 확장 규칙: 숙소/생활층을 나중에 추가할 수 있도록 세로 이동 기능의 최소 구조 확정

### 7순위 — UI/UX 통합 재설계 ← 현재

목표: 플립으로 화면을 교체하는 방식 대신, 가로로 긴 함선 내부를 카메라가 이동하는 구조로 전환한다.
기본 홈은 중앙 파일럿 라운지/브릿지로 두고, 좌측에는 확장형 격납고, 우측에는 관제실을 배치한다.
공작실은 격납고 팝업, 상점/고용은 관제실 팝업으로 호출한다.
상세 계획은 `docs/plans/2026-05-27-ux-redesign.md`를 기준으로 한다.
