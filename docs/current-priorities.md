# 현재 우선순위

## 현재 단계

직접 파견 + 자동 파견 기본 루프 모두 완성.  
파츠 개별 아이템 시스템 리팩터링(v3) + 격납고 조립 팝업/인벤토리 구현 완료.
PC 터미널 메뉴형 레이아웃 구현 완료.  
ShipCanvas (가로 파노라마 함선 캔버스) 구현 완료.  
항성지도 팝업 전면 재설계 완료 — 행성 슬롯 4행 그리드, 베이 팝업 슬라이드인, 우클릭 드래그 스크롤.  
직접 파견 클리커 UX 개선 완료 — "미션완료" / "함선복귀" 플로팅 텍스트.  
격납고 구역 (`HangarZone`) 구현 완료 — 4격납고×4베이 구조, 오버레이 팝업 방식 상세, 격납고 잠금/해금.  
세이브 v4 전환 완료 — hangar_groups 직렬화, v3→v4 마이그레이션 내장.

현재 작업은 **Living Ship Pivot** 진행 중.  
관제실/격납고의 기존 팝업 흐름은 유지하되, 내부 항목을 함선 생활 중심으로 재구성하는 단계다.

### 2026-05-29~30 작업 (게임플레이 컨텐츠 확장)
- 레거시 플립 패널 4종 제거 + `ship_canvas` / `hangar_zone` 코드 분리 리팩터링
- 클리커 강화 시스템: 파츠 상점에 **강화 탭** 추가 (클릭 데미지 / 자동공격 / 클릭 범위 / 연타 콤보)
- 클리커 생물 6종 + 이동 패턴 (좌우/상하/원/반사/8자/불규칙), 행성 난이도별 티어 매핑
- 파츠 드랍 시스템 (40% 확률) + 드랍 파츠 옵션 2개 부착 / 효과 적용 / 분해 환급

- 파일럿 생활 상태의 회복 우선순위: 침대 / 소파 / 식당 / 의무실 순서 (현재 stress≥70 recover > stress≥50 play > fatigue≥50 rest > mood≤40 eat)

- **자동 파견 크레딧 스케일 문제 (2026-05-31 분석, 변경 보류)**: T3 풀스펙 기준 자동 파견 1사이클 수익이 ~3,000 CR로 사실상 상한(무기 base 1,800 + 행성 CP 100~1,800). 행성 CP는 전 구간 약 18배만 증가하나 해금 비용은 ~10¹² 배 증가 → 후반(sector_n 이후)에는 다음 행성 해금까지 자동 파견 임무가 수십만~1억 회 필요, 크레딧 공급원으로 무의미. 직접 클리커(credit_per_kill ~2400배 스케일, CR/HP 효율 평탄 5.0)가 실질 성장 동력. 해결 옵션: ① 후반 행성 CP를 difficulty 비례로 공격적 스케일 ② 자동 파견=재료 특화로 의도 확정 ③ 무기 T4+/배율 확장. (해금 비용 배율 자체는 ×3.3~3.8로 일정 — cliff 없음)
- 관계 / 기억파편 시스템의 MVP 범위
