# 현재 우선순위

## 현재 단계

직접 파견 + 자동 파견 기본 루프 모두 완성.  
파츠 개별 아이템 시스템 리팩터링(v3) + 공작실 인벤토리 탭 구현 완료.  
PC 터미널 메뉴형 레이아웃 구현 완료.  
격납고: 좌측 슬라이드 인 상세 패널 재설계 완료 (뒤로가기 가림 버그 수정 포함).  
파견 관제 패널 전면 재설계 완료 (anchor Tween 2-state 레이아웃, 전체 슬롯 스크롤).  
함선 강화: 자동 공격 업그레이드 추가 완료.  
직접 파견 버그 수정 완료 (player_status 고착, 히스토리 스택 오염).  
행성 몬스터 수 ×10, 초기 자금 0 CR, Windows exe 빌드 완료.  

현재 작업은 **UI/UX 통합 재설계**다. 기존 플립 기반 패널 교체 구조를
`가로로 긴 함선 캔버스 + 카메라 이동` 구조로 바꾸는 방향으로 확정했다.
중앙 브릿지/파일럿 라운지, 좌측 격납고, 우측 관제실 3구역을 기본 구조로 둔다.
사운드 시스템은 이 구조 통합 이후에 붙인다.

## 논의 미완료 항목

- 관제실 UI/UX: 항성지도 팝업, 행성 상세, 슬롯/기체 선택, 파견 명령의 정보 우선순위를 확정
- 격납고 UI/UX: 베이 확장, 귀환 수령, 머신 관리, 공작실 팝업을 어떤 흐름으로 구성할지 확정
- 반복 조작 대응: 쉬리릭 이동 연출과 빠른 조작 편의 사이의 균형 확정
- 아래 데크 확장 규칙: 숙소/생활층을 나중에 추가할 수 있도록 세로 이동 기능의 최소 구조 확정

### 7순위 — UI/UX 통합 재설계 ← 현재

목표: 플립으로 화면을 교체하는 방식 대신, 가로로 긴 함선 내부를 카메라가 이동하는 구조로 전환한다.
기본 홈은 중앙 파일럿 라운지/브릿지로 두고, 좌측에는 확장형 격납고, 우측에는 관제실을 배치한다.
공작실은 격납고 팝업, 상점/고용은 관제실 팝업으로 호출한다.
상세 계획은 `docs/plans/2026-05-27-ux-redesign.md`를 기준으로 한다.
