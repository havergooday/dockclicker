# 구현된 기능

## 개요

이 문서는 현재 저장소에 실제로 반영된 기능, 구조, 문서를 정리한다.  
여기서 말하는 `구현된 기능`은 플레이 가능한 게임 기능뿐 아니라, 프로젝트 운영에 필요한 문서/구조/검증도 포함한다.

## 행성 시스템 구현 완료

| 행성 | 해금 비용 | 동시 출현 | 웨이브 총수 | CR/킬 | 적 HP |
|---|---|---|---|---|---|
| 섹터 A | 무료 | 2 | 5 | 10 | 2 |
| 섹터 B | 50 CR | 3 | 8 | 15 | 3 |
| 섹터 C | 200 CR | 4 | 12 | 25 | 5 |

- 파견 관제 패널에서 행성 선택 UI (버튼 3개)
- 미해금 행성 클릭 → 크레딧 차감 + 자동 해금 + 선택
- 크레딧 부족 시 버튼 비활성화

## PC 터미널 — 파츠 상점 구현 완료

- **클릭 데미지 강화**: 4단계 (20→50→100→200 CR), 최대 데미지 5
- **파츠 구매** (파일럿·몸체·무기·다리, 각 3티어, 소모성 아이템)
  - 파일럿 Lv1~3: 행성 해금 조건 적용 (Lv1=섹터A, Lv2=섹터B, Lv3=섹터C)
  - 몸체 Lv1~3: 파견 시간 +30/75/150초
  - 무기 Lv1~3: CR/s ×2/5/12
  - 다리 Lv1~3: 복귀 -5/12/25초
- **보유 파츠 탭**: 개별 아이템 슬롯 형태 인벤토리 (수량이 아닌 개별 행)

## 공작실 패널 구현 완료

- 보유 파츠 인벤토리 표시 (파일럿·몸체·무기·다리 개별 슬롯)
- 파견중 파일럿 별도 표시 (반투명 + "파견중" 태그)
- 귀환중 파일럿도 파견중으로 표시

## 격납고 — 자동 파견 슬롯 시스템 구현 완료

**슬롯 상태 흐름:** locked → empty → offline → on_mission → returning → returned → offline

- 슬롯 3개 (슬롯1 기본 해금, 슬롯2=300CR, 슬롯3=700CR 해금)
- 정사각형 버튼 그리드(3열) — 상태별 색상 구분
  - OFFLINE (회색): 클릭 → 파견 관제로 이동 (해당 슬롯 자동 선택)
  - 파견중 (빨간색): 비활성
  - 귀환중 (주황색): 비활성
  - 복귀완료 (초록색): 클릭 → 크레딧 수령 + 오프라인 복귀
- 빈 슬롯 클릭 → 머신 조립 서브뷰
  - 보유 파츠(몸체/무기/다리) 선택 후 조립 비용(CR) 차감
  - 조립 완료 → 해당 슬롯에 머신 배치
- 격납고 진입 시 항상 메인(슬롯 목록) 화면으로 리셋

## 파견 관제 패널 — 자동 파견 구현 완료

- **자동 파견 섹션**: 머신 선택 → 파일럿 선택 → 파견 지역 선택 → 출격
  - 파일럿 등급에 따라 접근 가능 행성 필터링 (Lv1=섹터A까지, Lv2=섹터B까지 등)
  - 출격 성공 시 토스트 알림 ("슬롯 N 파견 시작!")
  - 격납고에서 슬롯 선택 후 파견 관제 진입 시 해당 슬롯 자동 선택

## 자동 파견 타이머 시스템 구현 완료

- **임무 단계** (`on_mission`): 몸체 파츠 가치(초) 동안 진행
- **귀환 단계** (`returning`): 기본 30초 − 다리 파츠 감소값 (최소 5초)
- **복귀완료** (`returned`): 격납고에서 수동 수령 필요
  - 수령 시: 무기 CR/s × 임무 시간 크레딧 지급, 파일럿 인벤토리 반환, 슬롯 offline 복귀
- `GameState._process()` — 매 프레임 타이머 체크, 상태 자동 전환

## 브릿지 — 파견 현황 패널 구현 완료

- 브릿지 하단에 파견 현황 상시 표시
- 임무중 슬롯: 슬롯 번호·목적지 + 임무 완료까지 카운트다운 (MM:SS)
- 귀환중 슬롯: 주황색 + 복귀까지 카운트다운 (MM:SS)
- 파견 없음: 흐린 안내 문구

## 현재 상태 요약

- 직접 파견 + 자동 파견 기본 플레이 루프 모두 완성.
- 파츠 구매 → 조립 → 파견 → 귀환 → 수령 → 재투자 전체 사이클 동작.
- 저장/불러오기, 자동 재파견, 행성별 그래픽은 미구현.
